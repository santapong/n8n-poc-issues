How to read it: the blue line is the V8 heap from /metrics. The green line is the real container memory (via the Docker socket) — this is the one that reveals the JS Task Runner OOM on the worker, which the heap metric can't see. Upload a big file and watch n8n-main: if the blue/green lines stay flat, the web node is not going down.

Want the actual objects inside the heap (constructors, retained sizes, string/buffer contents) — not just totals? Take a full heap snapshot and open it in Chrome DevTools:
scripts/heap-snapshot.sh n8n-main → then Chrome ▸ chrome://inspect ▸ Load ▸ pick the file in ./heapsnapshots/. The per-space bar above is the live coarse view; the snapshot is the full object-level view.
